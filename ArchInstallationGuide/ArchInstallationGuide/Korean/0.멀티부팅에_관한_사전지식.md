## 멀티부팅이란 무엇인가?
멀티부팅이란 하나의 컴퓨터에서 윈도우와 리눅스 등 여러 운영체제(OS)를 설치하고, 부팅 시 실행할 운영체제를 선택하는 과정을 말한다. 각 운영체제는 일반적으로 별도의 파티션이나 물리적 드라이브에 설치되므로, 서로 간섭 없이 독립적으로 작동할 수 있다.


## 부트로더의 존재가 생소한 이유
부트로더는 리눅스에서만 사용되는 것이 아니라, 모든 운영체제가 실행하기 위해서 필수적으로 먼저 실행되어야 하는 프로그램이다. 윈도우와 맥OS가 다른 운영체제와의 기본적인 멀티부팅 기능을 제공하지 않기 때문에, 윈도우와 맥 사용자들은 매일 컴퓨터를 사용함에도 부트로더의 존재 자체를 모르는 경우가 많다. 멀티부팅 시스템에서는 GRUB과 같은 부트로더가 먼저 화면에 실행되어 어떤 운영체제를 실행할 것인지 묻는다. 만약 한 기기에서 여러 운영체제를 동시에 사용할 계획이라면, 안전하게 시스템을 유지하기 위해 부트로더에 대한 개념을 익히는 것을 추천한다. 


## 펌웨어란?
시중에서 접할 수 있는 PC는 바이오스(Basic Input Ouput System, BIOS) 또는 UEFI(Unified Extensible Firmware Interface) 라는 펌웨어가 메인보드에 내장된 채로 판매된다. 바이오스는 디스크 첫 부분에 쓰여진 boot sector를 읽어 운영체제를 불러온다. UEFI 는 바이오스보다 개선된 방식으로, EFI 파티션에 있는 .efi 부트로더를 읽어 실행한다. 바이오스와 UEFI 는 그 기능과 역할이 다르며, 최신 컴퓨터들은 대부분 UEFI 방식을 사용하지만 많은 기업에서 편의적으로 BIOS 라고 부른다. 컴퓨터의 전원 버튼을 누르면 가장 먼저 바이오스가 실행되며, 현재 사용 가능한 부트로더를 우선순위에 따라 실행한다.

**BIOS or UEFI (Firmware) -> Boot Loader -> Operating System**


## EFI 시스템 파티션이란?
UEFI로 동작하는 컴퓨터 시스템은 부트로더를 저장하는 EFI 시스템 파티션을 가진다. 파티션은 하나의 물리 저장 장치에 논리적으로 구역을 나눈 것이다. 펌웨어가 메인보드에 직접 설치되는 것과 다르게, EFI 파티션은 HD 또는 SSD 와 같은 보조기억장치에 저장된다. 윈도우의 디스크 관리 기능에서 현재 운영체제의 파티션 구조를 확인할 수 있다.

**예시 구조** : | EFI System partition | Reserved (Hidden) | C:\ | D:\ |


![disk_management](./Sources/disk_management.png)
- 맨 앞에 위치한 것이 EFI 파티션이며, 이 곳에 여러 부트로더가 저장된다.


## 펌웨어가 부트로더를 불러오는 원리
UEFI 시스템에서 컴퓨터가 켜지면, UEFI 가 등록된 부트로더를 우선순위에 따라 실행한다. 멀티부팅 시스템이라면 운영체제를 선택할 수 있는 화면이 나오며, 사용자는 부팅하고자 하는 운영체제를 선택한다. 만약 가능한 부트로더 실행 옵션이 존재하지 않는다면 Fallback 경로를 자동으로 실행하며, 기본적인 Fallback 경로는 "EFI/BOOT/bootx64.efi" 이다.

```
Power On  
  ↓  
UEFI Firmware  
  ↓  
Bootloader (GRUB)  
  ↓  
Linux
```

```
EFI (EFI partition)
 ├── BOOT
 │    └── bootx64.efi 
 └── Microsoft
```
- 이 Fallback 경로에 해당하는 부트로더의 동작은 실제 컴퓨터의 상황에 따라 다를 수 있다.

리눅스를 설치한 이후, 리눅스의 기본 부트로더(배포판에 따라 다를 수 있다)인 GRUB 이 EFI 파티션과 펌웨어의 부트로더 리스트에 추가된다.

```
EFI (EFI partition)
 ├── BOOT
 │    └── bootx64.efi 
 ├── GRUB
 │    └── grubx64.efi
 └── Microsoft
```


## 설치 이후 파일 구조
리눅스 설치 이후 리눅스 내에서 바라본 부팅 시스템 파일의 대략적인 구조는 다음과 같다. 

```
/boot
 ├── vmlinuz-linux         
 ├── initramfs-linux.img  
 ├── efi
 │ 	  └── EFI
 │ 	   	   ├── BOOT
 │ 		   │    └── bootx64.efi 
 │ 	       ├── GRUB
 │ 	       │    └── grubx64.efi
 │ 	       └── Microsoft
 └── grub
      └── grub.cfg         
```

대부분의 배포판에서 이러한 구조로 부트로더를 자동적으로 설치하기 때문에 사용자가 직접 만들 필요는 없으나, 부트로더를 커스텀하거나 부팅 과정에 문제가 생겼을 경우 수정해야 할 파일을 확인하기 위해서 다음과 같은 구조로 이루어짐을 확인하는 것이 좋다.

| In Windows             | In Linux | Meaning                    |
| ---------------------- | -------- | -------------------------- |
| EFI System Partition   | sda1     | EFI System Partition       |
| Reserved Space         | sda2     | Windows Reserved Partition |
| C Drive                | sda3     | Windows C:\ Drive          |
| Not Mounted in Windows | sda4     | Linux Root Partition       |
- 위 표는 윈도우가 설치되어있던 컴퓨터에 리눅스를 설치한 이후 파티션 구조의 예시이다.
